  • 什么是Git呢？

    Git是一个开源的分布式版本控制系统，用以有效、高速的处理从很小到非常大的项目版本管理。Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件。

  • 分布式与集中式的区别在哪里？

    对于分布式而言，开发者可以把代码提交到本地，他可以把远端服务器上的仓库克隆到本地，在本地进行修改再提交到远端仓库中

  • Git的优点

    1.  分布式开发强调个体

    2.  可以有效的解决多个开发者关于项目修改等冲突问题

    3.  支持离线工作

    4.  速度快且灵活

  • Git的缺点

    所谓金无足赤，人无完人嘛，对于Git也是一样的，提供了许多的便利的同时也会带来一些缺陷:

    代码的保密性差，一旦你从远端克隆到本地后，你将可以看见所有的代码及其版本信息了（小主个人认为这是最主要的缺陷了）

  • 接下来讲讲Git的工作原理

    假设在你的本地已经有了克隆好的仓库，此时你的本地仓库是由git维护的“三棵树”组成的：第一个是你的工作目录，它持有实际文件；第二个是缓存区（index）它像一个临时区域，临时保存你的改动；最后是（HEAD）指向你最近一次提交的结果

  • 接下来讲一些Git的具体用法，（本来小主想把git的命令都列出来的，但是想想这并没有什么卵用啊，还是实际点举例并说明靠谱点）

    假如你在本地机器上没有自己的本地仓库，可是又想把自己写好的项目push到远端仓库里，可以执行以下操作：

    git init      //表示在当前目录中生成本地的git管理

    git add -A      //表示添加所有的文件

    git commit -m "******"      //表示对这次提交的注释

    git git remote add origin master https:*******      //将要上传到远程仓库的地址

    git remote rm origin      //若上一步执行失败后输入再重复上一步操作

    git push -u origin master      //输入完后敲回车键便会出现要求你输入用户名的提示，用户名输入完后敲回车便出现提示输入用户密码的提示了
